# 微博热搜分析器 (Weibo Hot Analyzer)

## 静默执行协议 (Silent Execution Protocol)

> **⚠️ 原子性执行规则 - 必须严格遵守**
>
> 1. **禁止中途确认**：不要询问"是否继续"、"需要更多信息吗"等问题
> 2. **一次性完成**：获取数据后立即完成全部分析和报告生成
> 3. **使用默认值**：未指定参数时使用默认值，不要询问
> 4. **静默处理错误**：遇到问题记录日志并继续，不要中断流程

## 工作流程

执行微博热搜分析时，按以下步骤一口气完成：

1. **获取数据**：运行 `scripts/fetch_weibo_hot.py` 获取热搜数据
2. **分析热搜**：对 Top 10 逐条进行深度分析
3. **生成报告**：同时生成 HTML 和 Markdown 双格式报告
4. **输出文件**：保存到 `weibo-hot-reports/` 目录并提供链接

## 脚本使用

### 获取微博热搜数据

```bash
python scripts/fetch_weibo_hot.py
```

脚本会返回 JSON 格式的热搜数据，包含：
- `title`: 热搜标题
- `hot_value`: 热度值
- `category`: 分类标签
- `url`: 话题链接

## 分析维度

对每条热搜进行以下分析：

| 维度 | 说明 | 输出格式 |
|------|------|----------|
| **分类** | 娱乐/社会/科技/体育/财经/其他 | 单个标签 |
| **核心摘要** | 提炼事件核心，50-100字精华 | 短段落 |
| **关键要点** | 3-5个核心信息点 | 列表 |
| **商业化洞察** | 可商业化的机会点 | 描述或"无明显商业化机会" |

## 商业化分析框架

分析商业化机会时，从以下角度切入：

1. **品牌营销机会**：是否适合品牌借势营销
2. **电商选品机会**：是否有相关产品可推广
3. **内容创作机会**：是否适合自媒体内容创作
4. **广告投放机会**：是否有广告价值
5. **IP/周边机会**：是否有 IP 衍生价值

如果所有角度都没有明显机会，输出：**"无明显商业化机会"**（不要询问用户）

## 输出规格

### 文件命名

```
weibo-hot-{YYYY-MM-DD-HH-mm}.html
weibo-hot-{YYYY-MM-DD-HH-mm}.md
```

### 输出目录

在当前工作目录创建 `weibo-hot-reports/` 文件夹保存报告。

### HTML 报告设计规范

使用 2026 前沿浅色设计：
- 配色：主色调 #667eea（渐变紫蓝），背景 #f8fafc
- 字体：Inter, "PingFang SC", sans-serif
- 布局：响应式卡片布局，圆角 16px
- 动效：微动画过渡，悬停效果
- 特效：玻璃拟态（Glassmorphism）细节

参考 `assets/report-template.html` 中的模板。

## 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| API 调用失败 | 重试1次，仍失败则输出错误信息到报告 |
| 单条热搜解析失败 | 标记"解析失败"，继续处理其他条目 |
| 文件写入失败 | 在控制台输出内容，提示保存失败原因 |
